# Negative UI / Bad Website Link Database

Use these as negative references only. Do not clone them. Study what fails: clutter, weak hierarchy, bad responsiveness, unreadable type, dated layout, confusing navigation, deceptive flows, or hostile interaction patterns.

Some links may change, break, autoplay, contain offensive content, or show intentionally ugly/legacy design. Open questionable ones in a safe browser profile.

## Agent Study Prompt

Use this negative link database to learn what not to do.

For each site or example, extract:

1. What makes the hierarchy fail
2. What makes the navigation confusing
3. What makes the typography hard to use
4. What makes the color system unpleasant or inaccessible
5. What makes the layout feel cluttered, dated, or careless
6. What interaction patterns feel hostile or misleading
7. What breaks on mobile or small screens
8. What content is missing, vague, duplicated, or overwhelming
9. What visual decisions would be unacceptable in a modern product UI
10. What one lesson should be encoded as a design rule

Do not copy the negative examples. Convert the failure modes into constraints for a better original UI.
